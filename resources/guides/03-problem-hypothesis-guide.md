# 문제 가설 정의 가이드 (공연 기획)

## 목적

리서치 및 시장·관객 분석 데이터를 기반으로 관객과 시장이 겪는 핵심 문제 가설과 근본원인을 도출하여,
공연이 해결해야 할 본질적 문제를 명확히 정의함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 작품 리서치 | `output/{작품명}/01-research.md` | 유사 공연 사례에서 관객 Pain Points 파악 |
| 시장·관객 분석 | `output/{작품명}/02-market-analysis.md` | 타겟 관객 프로파일·시장 동향에서 문제 도출 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|
| 문제 가설 | `output/{작품명}/03-problem-hypothesis.md` |

## 방법론

### 1. 문제 가설과 근본원인 도출

공연 기획이 해결하고자 하는 관객·시장의 핵심 문제 3개를 도출하고, 각각 5WHY로 근본원인을 분석함.

> **문제 작성 원칙**: 솔루션(어떤 공연을 만들 것인가)이 아닌 관객·시장이 겪는 현상 문제로 작성.
> 예시: "관객이 공연 정보를 접하기 어렵다" ✓ / "뮤지컬을 만들어야 한다" ✗

#### 문제 유형 예시 (공연 도메인)

| 유형 | 예시 문제 |
|------|---------|
| **관객 경험** | 공연 관람 후 감동·여운이 부족하다 |
| **접근성** | 특정 연령대/계층의 공연 접근성이 낮다 |
| **시장 공백** | 특정 장르·테마의 콘텐츠 공급이 부족하다 |
| **경쟁·차별화** | 유사 공연과의 차별점이 없어 선택받지 못한다 |
| **인지도** | 타겟 관객층에게 공연이 알려지지 않는다 |

#### 5WHY 작성 형식

| 단계 | 질문 및 응답 |
|:----:|-------------|
| **WHY1** | 왜 {현상}인가? → {응답} |
| **WHY2** | 왜 {WHY1 응답}인가? → {응답} |
| **WHY3** | 왜 {WHY2 응답}인가? → {응답} |
| **WHY4** | 왜 {WHY3 응답}인가? → {응답} |
| **WHY5** | 왜 {WHY4 응답}인가? → {응답} |

- WHY1은 현상문제에 대한 질문 + 응답으로 작성
- WHY2~WHY5는 이전 WHY 응답에 대한 질문 + 응답으로 작성

#### 다층적 근본원인 검토 (공연 도메인 적용)

근본원인을 아래 4가지 측면으로 검토하여 가장 중요한 측면 하나로 근본원인을 식별함:

| 측면 | 공연 도메인 검토 항목 |
|------|---------------------|
| **관객 심리적** | 관람 동기·감정 반응·습관·저항감·기대 수준 |
| **시장·구조적** | 공연 시장 구조·배급·티켓 유통·제작비 구조·경쟁 환경 |
| **맥락적** | 문화 트렌드·사회적 분위기·시즌성·타이밍·지역 특성 |
| **커뮤니케이션** | 홍보 방식·마케팅 채널·입소문·리뷰 생태계 |

#### 핵심 근본원인 식별 형식

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 관객 심리적 | {근본원인} | ★★★★☆ |
| 시장·구조적 | {근본원인} | ★★★☆☆ |
| 맥락적 | {근본원인} | ★★★★★ |
| 커뮤니케이션 | {근본원인} | ★★★☆☆ |

### 2. 비즈니스 가치 도출

근본원인을 해소하여 문제를 해결했을 때 관객과 제작사가 얻는 비즈니스 가치를 각각 3개 이하로 도출함.

```markdown
## 관객 측면의 가치
- {가치 1}
- {가치 2}
- {가치 3}

## 제작사 측면의 가치
- {가치 1}
- {가치 2}
- {가치 3}
```

### 정량 기준
- 핵심 문제: 3개
- 5WHY: 문제당 5단계
- 근본원인 검토: 4가지 측면
- 비즈니스 가치: 관객/제작사 각 3개 이하

## 주의사항

- 문제는 관객·시장 관점에서 작성 (제작자 관점 아님)
- 솔루션(공연 컨셉)은 포함하지 않음
- 5WHY는 최소 3단계 이상, 표면 증상이 아닌 구조적 원인까지 탐색
- 01-research.md의 유사 공연 사례, 02-market-analysis.md의 시장 데이터를 근거로 활용
