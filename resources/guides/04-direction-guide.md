# 방향성 정의 가이드 (킹핀 문제 + 공연 방향성)

## 목적

도출된 문제 가설 중에서 가장 핵심적인 킹핀 문제를 선정하고,
이를 해결하기 위한 공연 기획의 명확한 방향성을 Needs Statement로 정의함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 문제 가설 | `output/{작품명}/03-problem-hypothesis.md` | 검증된 문제 리스트업 및 인과관계 분석 |
| 시장·관객 분석 | `output/{작품명}/02-market-analysis.md` | 타겟 관객 유형 정의 및 킹핀 선정 근거 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|
| 킹핀 문제 + 공연 방향성 | `output/{작품명}/04-direction.md` |

## 방법론

### 킹핀 문제 개념

볼링에서 스트라이크를 치기 위해 반드시 쓰러뜨려야 하는 5번 핀처럼,
이 문제를 해결하면 나머지 문제들이 연쇄적으로 해결되는 가장 핵심적인 문제임.

공연 기획 맥락에서는: 이 공연이 집중적으로 해소해야 할 하나의 본질적인 관객·시장 문제.

### 5가지 선정 기준 (각 5점)

| 기준 | 공연 도메인 핵심 질문 | 평가 포인트 |
|------|-------------------|------------|
| **영향력 (Impact)** | 이 문제를 해결하면 다른 문제도 함께 해결되는가? | 연쇄 해결 문제의 수와 중요도 |
| **빈도 (Frequency)** | 관객이 이 문제를 얼마나 자주 겪는가? | 매 관람/가끔/특정 상황에서만 |
| **심각도 (Severity)** | 관객에게 미치는 고통·실망의 정도는? | 재관람 포기·비추천·부정적 경험 |
| **근본성 (Root Cause)** | 다른 문제들의 원인이 되는가? | 5WHY 분석 결과, 인과관계 |
| **해결가능성 (Feasibility)** | 이 공연으로 현실적으로 해결 가능한가? | 제작 역량·예산·일정·기술 |

### 평가 기준 (Rubric)

| 점수 | 영향력 | 빈도 | 심각도 | 근본성 | 해결가능성 |
|------|--------|------|--------|--------|-----------|
| 1 | 극히 일부 관객에게만 영향 | 매우 드물게 발생 | 사소한 불편 | 표면적 증상 | 현재 역량으로 불가능 |
| 2 | 일부 관객의 일부 경험에 영향 | 특정 상황에서만 | 불편하지만 대안 있음 | 하위 문제 1개의 원인 | 대규모 자원 필요 |
| 3 | 다수 관객의 일부 경험에 영향 | 가끔 발생 | 관람 만족도 저하 | 하위 문제 2~3개의 원인 | 노력 필요하나 가능 |
| 4 | 다수 관객의 핵심 경험에 영향 | 자주 발생 | 재관람 의사 감소 | 다수 문제의 공통 원인 | 충분히 실현 가능 |
| 5 | 전체 관객의 핵심 경험에 영향 | 항상 발생 | 비추천·이탈 유발 | 거의 모든 문제의 근본 | 즉시 실현 가능 |

### 분석 방법

#### Step 1: 문제 리스트업
| 문제 ID | 문제 설명 | 근본원인 요약 |
|---------|-----------|-------------|
| P1 | {문제} | {근본원인} |

#### Step 2: 문제 간 인과관계 맵핑
```
P1 (근본원인) → P2 (중간문제) → P3 (표면문제)
```

#### Step 3: 5가지 기준 평가
| 문제 ID | 영향력 | 빈도 | 심각도 | 근본성 | 해결가능성 | 총점 |
|---------|--------|------|--------|--------|------------|------|

#### Step 4: 킹핀 문제 선정 및 근거 작성

### Needs Statement 작성 (공연 방향성)

#### 구조
```
'{타겟 관객유형}는 {목적}을 위하여 {원하는 공연 경험}이 필요하다.'
```

#### 작성 원칙

| 원칙 | 설명 |
|------|------|
| **관객 경험 중심** | 특정 공연 형식이 아닌 관객이 원하는 경험으로 표현 |
| **목적 명확화** | 킹핀 문제의 근본원인을 해소하는 것을 목적으로 설정 |
| **원하는 것 정의** | 장르·형식이 아닌 경험과 가치로 표현. 공연의 큰 방향성 정의 |
| **하나의 문장** | 단 1개의 문장으로 표현 |

#### 예시 (뮤지컬 기획)

```
'일상에 지친 직장인은 현실 탈출과 정서적 치유를 위하여
화려한 볼거리보다 진심으로 공감되는 이야기와 감동을 주는 공연 경험이 필요하다.'
```

#### 검증 체크리스트
- [ ] 타겟 관객유형이 명확하게 정의되어 있는가?
- [ ] 목적이 킹핀 문제의 근본원인 해결과 연결되어 있는가?
- [ ] 원하는 것이 장르/형식이 아닌 경험과 가치로 표현되었는가?
- [ ] 하나의 문장으로 명확하게 표현되었는가?
- [ ] 특정 솔루션(작품명, 공연 방식)이 아닌 방향성을 제시하는가?

## 품질 기준

### 완료 체크리스트
- [ ] 5가지 기준(영향력/빈도/심각도/근본성/해결가능성) 평가 완료
- [ ] 킹핀 문제 1개 선정 (선정 근거 포함)
- [ ] 문제 간 인과관계 분석 포함
- [ ] Needs Statement 1문장 작성
- [ ] Needs Statement가 특정 작품/형식이 아닌 경험으로 표현됨
- [ ] 03-problem-hypothesis.md 데이터 기반 결정

### 정량 기준
- 평가 기준: 5가지 (각 5점 만점)
- 킹핀 문제: 1개
- Needs Statement: 1문장

## 주의사항

- 킹핀 문제는 객관적 데이터 기반 결정 (감정적 판단 지양)
- Needs Statement는 솔루션(공연 컨셉)이 아닌 방향성 제시
- 킹핀 문제 해결이 다른 문제들의 연쇄 해결로 이어지는지 검증
- 타겟 관객은 02-market-analysis.md의 관객 프로파일 참조
