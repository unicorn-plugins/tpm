# 아이디어 발상 가이드 (공연 컨셉 후보 도출)

## 목적

문제 가설과 공연 방향성을 기반으로 창의적인 공연 컨셉 아이디어를 다각도로 생성하고,
유사한 아이디어를 수렴하여 실질적인 공연 컨셉 후보군을 도출함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 작품 리서치 | `output/{작품명}/01-research.md` | 유사 공연 사례 → Steal & Synthesize 대상 |
| 시장·관객 분석 | `output/{작품명}/02-market-analysis.md` | 시장 기회·경쟁 분석 → 아이디어 영감 |
| 문제 가설 | `output/{작품명}/03-problem-hypothesis.md` | 해결할 문제의 범위와 근본원인 |
| 공연 방향성 | `output/{작품명}/04-direction.md` | Needs Statement를 아이디어 발상 출발점으로 활용 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|
| 공연 컨셉 후보 | `output/{작품명}/05-concept-candidates.md` |

## 방법론

### 브레인스토밍 원칙

- **양이 질을 낳는다**: 최대한 많은 아이디어 생성 (자기 검열 금지)
- **관객 중심**: 제작 편의가 아닌 관객의 경험과 감동 관점에서 발상
- **다양성 확보**: 예산·규모·장르·형식·대상 연령대를 다양하게 탐색
- **Needs Statement 연결**: 방향성에서 제시한 관객 경험 가치를 달성하는 방향으로

### 아이디어 구분

| 구분 | 정의 | 수량 |
|------|------|------|
| **Big Idea** | 제작 자원은 많이 필요하지만 성공 시 관객 임팩트가 큰 컨셉 | 3개 |
| **Little Win Idea** | 비교적 빠르고 현실적으로 실현 가능한 컨셉 | 2개 |
| **Crazy Idea** | 파격적이고 실험적이나 성공 시 차별화 임팩트가 극대화되는 컨셉 | 1개 |

### 아이디어 발상 기법

#### 1. SCAMPER 기법 (공연 도메인 적용)

기존 공연 형식이나 아이디어를 7가지 관점에서 변형하여 새로운 컨셉을 창출함.

| 기법 | 공연 도메인 핵심 질문 | 적용 예시 |
|------|-------------------|---------|
| **S - Substitute (대체)** | 기존 배우·무대·음악을 다른 것으로 대체하면? | 그림자극 → 홀로그램으로 대체 |
| **C - Combine (결합)** | 두 장르·소재·형식을 결합하면? | 연극 + 라이브 음악 결합 |
| **A - Adapt (적용)** | 다른 분야의 성공 공식을 공연에 적용하면? | 게임의 몰입 구조를 관극에 적용 |
| **M - Modify (수정)** | 규모·시간·공간을 변형하면? | 소극장 → 야외 / 120분 → 60분 |
| **P - Put to other uses (다른 용도)** | 이 공연을 다른 관객/목적에 활용하면? | 성인극 → 가족 버전으로 확장 |
| **E - Eliminate (제거)** | 불필요한 요소를 제거하면? | 자막·막간 없는 몰입 공연 |
| **R - Reverse (역전)** | 공연 구조를 뒤집거나 역발상하면? | 결말부터 시작하는 역순 서사 |

**SCAMPER 활용 가이드:**
- Big Idea: **Adapt**, **Combine**, **Reverse** 관점 적극 활용
- Little Win Idea: **Eliminate**, **Modify** 관점으로 현실화
- Crazy Idea: **Substitute**, **Put to other uses** 관점으로 파격화

#### 2. Steal & Synthesize (타 분야 벤치마킹)

다른 분야의 성공 사례에서 핵심 메커니즘을 추출하고 공연에 재적용함.

##### Steal (벤치마킹 영역)

| 분야 | 공연에 적용 가능한 핵심 메커니즘 |
|------|-------------------------------|
| **게임** | 관객 참여·선택지·레벨 진행·리워드 구조 |
| **OTT/영상** | 몰입 서사·클리프행어·시즌제·예고편 효과 |
| **전시·예술** | 인터랙티브 체험·공간 연출·몰입형 환경 |
| **테마파크** | 스토리텔링 공간·체험형 서사·반복 방문 유도 |
| **이머시브 경험** | 관객 이동·개인화 동선·소규모 집중 |
| **SNS/바이럴** | 공유 욕구 유발·포토존·해시태그 문화 |
| **교육** | 지식 전달 + 감동·워크숍형·학습 여정 |

##### Synthesize (재합성 프로세스)
1. 유사점 찾기: 벤치마킹 사례의 성공 메커니즘과 우리 공연의 공통점
2. 차이점 인식: 공연 예술만의 제약과 강점
3. 변형 설계: 핵심 메커니즘을 공연 맥락에 맞게 변형
4. 검증: 타겟 관객에게 실제로 유효한가?

### 참여 에이전트 및 관점

| 에이전트 | 발상 관점 |
|---------|---------|
| market-analyst | 관객·시장 매력도 / 유사 공연 대비 차별화 / 마케팅 가능성 |
| proposal-writer | 공연 기획 완성도 / 서사·구성 / 실현 가능성 |

### 아이디어 발상 → 수렴 프로세스

1. **발산**: market-analyst, proposal-writer 각자 6개 아이디어 도출 (병렬)
2. **수렴**: proposal-writer가 전체 12개 아이디어의 유사도 평가 후 합치기

### 유사도 평가 기준 (수렴 단계)

- **컨셉 유사도** (가중치 70%): 해결하려는 관객 문제와 제공 경험 가치의 유사성
- **형식 유사도** (가중치 30%): 공연 형태·장르·연출 방식의 유사성
- **합치기 기준**: 종합 유사도 0.7 이상일 때만 합침

### 유사도 평가 형식

| 아이디어 A | 아이디어 B | 컨셉 유사도 | 형식 유사도 | 종합 유사도 | 합치기 여부 |
|-----------|-----------|-----------|-----------|-----------|----------|

## 주의사항

- 초기 발상 단계에서 자기 검열 금지
- 제작 편의가 아닌 관객 경험 관점에서 발상
- 01-research.md의 유사 공연 레퍼런스를 Steal & Synthesize에 적극 활용
- 유사도 0.7 미만인 아이디어를 강제 합치지 않음
- 각 후보는 04-direction.md의 Needs Statement와 연결 확인
