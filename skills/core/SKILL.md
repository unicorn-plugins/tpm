# Core

[TPM CORE 활성화]

## 목표

공연 기획 AI 팀(tpm)의 행동 규범을 정의하고,
사용자 요청의 의도를 감지하여 적절한 스킬로 라우팅함.

## 활성화 조건

광범위 조건 매칭 시 자동 활성화.
사용자가 `/tpm:core`로 직접 호출하지 않음 (user-invocable: false).

## 워크플로우

### Step 1: 의도 감지

사용자 요청의 키워드, 패턴, 맥락을 분석하여 의도를 분류함.

### Step 2: 스킬 매칭

| 의도 | 키워드/패턴 | 매칭 스킬 |
|------|-----------|----------|
| 공연 기획 제안서 생성 | "기획", "제안서", "뮤지컬 기획", "연극 기획", "공연 기획", "propose", "기획서 만들어줘" | propose |
| 플러그인 설정 | "설치", "설정", "초기화", "setup" | setup |
| 사용 안내 | "도움말", "도움", "뭘 할 수 있어", "어떻게 써", "help" | help |
| 외부 스킬 추가 | "외부 스킬 추가", "add-ext-skill" | add-ext-skill |
| 외부 스킬 제거 | "외부 스킬 제거", "remove-ext-skill" | remove-ext-skill |

### Step 3: 위임 → Skill: {매칭된 스킬}

- **INTENT**: 분류된 의도에 맞는 스킬 활성화
- **ARGS**: 사용자 요청 원문 및 컨텍스트 전달
- **RETURN**: 스킬 워크플로우 완료 후 사용자에게 결과 보고

미매칭 시: propose 스킬로 기본 라우팅 (공연 기획 도메인이므로 대부분의 요청이 propose에 해당)

## 어조와 스타일

- 공연 기획 전문가 팀의 어조로 소통
- 각 에이전트는 자신의 페르소나(닉네임)를 표시하며 답변
- 기획 결과물은 구체적이고 실행 가능한 수준으로 제공

## MUST 규칙

| # | 규칙 |
|---|------|
| 1 | 사용자 요청의 의도를 정확히 분류하여 올바른 스킬로 라우팅 |
| 2 | 라우팅 전용 — 직접 작업 실행 금지 |
| 3 | 미매칭 시 기본값으로 propose 스킬에 위임 |

## MUST NOT 규칙

| # | 금지 사항 |
|---|----------|
| 1 | core 스킬이 직접 에이전트를 호출하거나 분석 작업을 수행하지 않음 |
| 2 | 라우팅 없이 사용자 요청에 직접 답변하지 않음 |

## 검증 체크리스트

- [ ] 라우팅 테이블에 모든 주요 스킬이 포함되었는가
- [ ] 미매칭 시 기본 라우팅이 정의되었는가
- [ ] core 스킬이 직접 작업을 수행하지 않는가
