# Remove External Skill

[REMOVE-EXT-SKILL 활성화]

## 목표

tpm 플러그인에서 기존 외부호출 스킬(ext-{대상플러그인})을 안전하게 제거함.
사용자 최종 확인 후 스킬 디렉토리, commands 진입점, help 업데이트를 수행함.

## 활성화 조건

사용자가 `/tpm:remove-ext-skill` 호출 시.

## 워크플로우

### Step 1: 기존 ext-{} 스킬 목록 조회 (`ulw` 활용)

`skills/` 디렉토리에서 `ext-` 접두사 하위 디렉토리 탐색.
ext-{} 스킬이 0개이면:
- "제거할 외부호출 스킬이 없습니다." 안내 후 종료

발견된 ext-{} 스킬 목록을 사용자에게 표시.

### Step 2: 제거할 스킬 선택

사용자에게 제거할 ext-{대상플러그인} 스킬 선택 요청.
선택된 스킬의 SKILL.md를 읽어 스킬 정보(name, description) 표시.
"정말 제거하시겠습니까? (예/아니오)" 최종 확인.
사용자가 취소하면 즉시 중단.

### Step 3: ext-{대상플러그인} 스킬 디렉토리 삭제 (`ulw` 활용)

```bash
rm -rf skills/ext-{대상플러그인}/
```
삭제 성공 여부 확인.

### Step 4: commands/ 진입점 삭제 (`ulw` 활용)

```bash
rm -f commands/ext-{대상플러그인}.md
```
파일 미존재 시 무시.

### Step 5: help 스킬 업데이트 (`ulw` 활용)

`skills/help/SKILL.md`의 슬래시 명령 테이블에서 해당 행 제거.
제거 완료 메시지: "ext-{대상플러그인} 외부호출 스킬이 제거되었습니다."

## 명령어

| 명령 | 설명 |
|------|------|
| `/tpm:remove-ext-skill` | 외부호출 스킬 제거 시작 |

## MUST 규칙

| # | 규칙 |
|---|------|
| 1 | 삭제 전 반드시 사용자 최종 확인을 받을 것 |
| 2 | ext-{} 접두사가 아닌 스킬(setup, help, add-ext-skill, remove-ext-skill 등)은 제거 대상에서 제외 |
| 3 | help 스킬의 명령 테이블에서 해당 행만 정확히 제거 |

## MUST NOT 규칙

| # | 금지 사항 |
|---|----------|
| 1 | ext-{} 접두사가 아닌 스킬 디렉토리를 삭제하지 않음 |
| 2 | 사용자 확인 없이 삭제를 수행하지 않음 |
| 3 | help 스킬의 명령 테이블 구조(헤더, 구분선)를 훼손하지 않음 |

## 검증 체크리스트

- [ ] ext-{} 스킬 0개일 때 조기 종료가 동작하는가
- [ ] 삭제 전 사용자 최종 확인 단계가 존재하는가
- [ ] skills/ext-{대상플러그인}/ 디렉토리가 완전히 삭제되는가
- [ ] commands/ext-{대상플러그인}.md 파일이 삭제되는가
- [ ] help 스킬의 명령 테이블에서 해당 행이 제거되는가
- [ ] 다른 스킬/명령에 부수효과가 없는가
